data-prep-to-quick-estimate loop economists use Stata for: import, clean,

\section{Introduction}

\subsection{What \texttt{tea} is}

\texttt{tea} is a C17 program that runs as either an interactive REPL or
a batch processor for do-files.  It does the data manipulation and
econometrics that most applied economists do most days — and nothing
else.  No plotting, no Bayesian inference, no machine learning.  For
those, escape to Python or R; see \S\ref{sec:escape}.

The design priority is \emph{Stata fidelity in the overlapping subset}.
A do-file that runs in \texttt{tea} should run in Stata with the same
results, modulo a handful of small documented differences.  Where
Stata's behavior includes a footgun (\texttt{if x > 5} includes missing
values, for instance, because missing sorts above every number),
\texttt{tea} keeps the footgun.  That choice is deliberate: it makes
\texttt{tea} a drop-in for the data-prep phase, even if you eventually
move to Stata or R for the final analysis.

\subsection{What \texttt{tea} is not}

\texttt{tea} v\teaversion\ does \emph{not} implement: graphics of any
kind, exact-ML ARIMA (we use conditional likelihood), seasonal ARIMA,
GARCH, mixed-effects models, survival analysis, structural breaks, VAR,
VECM, cointegration tests, Bayesian inference, or machine learning.  It
also does not implement Stata's full programming language — \texttt{tea}
has macros, loops, and \texttt{capture}, but no \texttt{program define},
no \texttt{syntax} parser, no Mata.

The \texttt{shell} command makes it easy to bridge to Python or R when
you need any of the above.  See \S\ref{sec:escape}.

\subsection{Design choices, locked}

\begin{itemize}
\item \textbf{Storage} is one of two types: \texttt{double} (numeric, with
the full Stata missing-value algebra) or string.  Stata's \texttt{byte},
\texttt{int}, \texttt{long}, \texttt{float} are cosmetic on read and
compressed back on write of \texttt{.dta} files.
\item \textbf{Multiple frames} are supported.  Each has its own sort
state and panel/time settings.
\item \textbf{\texttt{by:} is strict}.  It errors if the data isn't
sorted; \texttt{bysort} sorts first.
\item \textbf{\texttt{reshape} is in-place}.  Use \texttt{frame copy}
first if you want to keep the original.
\item \textbf{\texttt{m:m} merges are rejected}.  They are almost never
what the user actually wants.
\item \textbf{C17}, no novelties.  Toolchain stability over
newer-standard ergonomics.
\end{itemize}

\section{Installation}

\subsection{Build from source}

You will have received a tarball, \texttt{tea-v\teaversion.tar.gz}.

\begin{lstlisting}[language=bash]
tar xzf tea-v1.0.tar.gz
cd tea
make
\end{lstlisting}

The build produces \texttt{./tea} in the project root.  Move it
somewhere on your \texttt{\$PATH} (e.g.\ \texttt{\textasciitilde/bin})
or invoke directly with \texttt{./tea}.

\subsection{Dependencies}

You need a C17 compiler (\texttt{gcc} 8+ or \texttt{clang} 7+) and the
following libraries with their development headers:

\begin{center}
\begin{tabular}{ll}
\toprule
Library & Why \\
\midrule
\texttt{libreadline} & line editing, history, tab completion in the REPL \\
\texttt{libopenblas} & matrix kernels \\
\texttt{liblapacke} & LAPACK C interface (regression, ARIMA, IV) \\
\texttt{libgsl} & distributions for $p$-values and confidence intervals \\
\texttt{libreadstat} & Stata \texttt{.dta} (and SAS / SPSS) I/O \\
\bottomrule
\end{tabular}
\end{center}

On Debian / Ubuntu:

\begin{lstlisting}[language=bash]
apt install libreadline-dev libopenblas-dev liblapacke-dev \
            libgsl-dev libreadstat-dev
\end{lstlisting}

On macOS (Homebrew):

\begin{lstlisting}[language=bash]
brew install readline openblas lapack gsl readstat
\end{lstlisting}

Run \texttt{make check-deps} after installing to verify every required
header is reachable.  If any dependency is missing, that target reports
which one and the install command to use.

\subsection{Sanity check}

\begin{lstlisting}[language=bash]
./tea --version          # prints "tea 1.0 -- tiny econometric assistant"
./tea tests/demo.do      # batch: runs a short demo (no errors expected)
make test                # full regression suite, 39 tests
\end{lstlisting}

If \texttt{make test} reports \texttt{tea regression: 39/39 passed},
your build is correct.

\subsection{Running interactively}

Start the REPL by invoking \texttt{tea} with no arguments:

\begin{lstlisting}[language=bash]
$ tea
tea 1.0 - tiny econometric assistant
type 'help' for commands, Ctrl-D to exit.
. _
\end{lstlisting}

History is persistent at \texttt{\~/.tea\_history} (capped at 2000 lines).
Tab completes command names at the start of a line and variable names
elsewhere.  \texttt{help} lists commands, \texttt{help CMD} shows the
syntax for one.

\section{Your first \texttt{tea} session}\label{sec:quickstart}

Let's load a small panel dataset, look at it, and run a regression.
Save the following as \texttt{quickstart.do}:

\begin{lstlisting}
* quickstart.do
import delimited "tests/WEO.csv", clear
rename INDICATOR_ID indicator_id
keep if indicator_id == "NGDP_RPCH"
rename COUNTRY_ID country_id
keep country_id v*
reshape long v, i(country_id) j(year)
rename v growth
keep if !missing(growth)

* Restrict to G7 countries for tractable output
keep if inlist(country_id, "USA", "DEU", "GBR", "FRA", "ITA", "JPN", "CAN")

* Convert string country codes to numeric so we can use factor vars
encode country_id, gen(cid)
xtset cid year

summarize growth
xtdescribe

* Fixed-effects regression of growth on its own lag
xtreg growth L.growth, fe
\end{lstlisting}

Run it:

\begin{lstlisting}[language=bash]
tea quickstart.do
\end{lstlisting}

This loads the IMF \emph{World Economic Outlook} CSV (shipped with the
distribution), pivots it from wide to long, restricts to G7 countries,
sets up the panel structure with \texttt{xtset}, and runs a fixed-effects
regression.  Output looks like Stata's; if you've used Stata before,
nothing here should surprise you.

A few things to notice:

\begin{itemize}[leftmargin=*]
\item \texttt{rename}, \texttt{keep if}, \texttt{reshape long} all work
the same as in Stata.
\item \texttt{encode} maps string country IDs to consecutive integers
1, 2, 3, $\ldots$ and attaches a value label so \texttt{list cid} still
shows ``USA'', ``DEU'', etc.
\item \texttt{xtset cid year} declares the panel structure.  After this,
\texttt{L.growth} means ``growth lagged one year, gap-aware within
country.''
\item \texttt{xtreg ..., fe} runs fixed effects (within transformation),
with classical SEs by default.  Add \texttt{vce(robust)} for HC1 or
\texttt{vce(cluster cid)} for clustered SEs.
\end{itemize}

\section{The do-file language}

A do-file is a sequence of \emph{commands}, one per line, with the
following syntax skeleton:

\begin{lstlisting}
[prefix:] command [varlist] [if exp] [in range] [weight] [, options]
\end{lstlisting}

\subsection{Comments}

Four comment forms are supported:

\begin{lstlisting}
* Whole-line comment starting with asterisk
// Inline comment to end of line
/* Block comment
   spanning multiple lines */
generate x = 1 ///
       + 2     // line continuation: combine the next line with this one
\end{lstlisting}

\subsection{Macros}

Local macros are referenced as ``\verb|`name'|'' (back-tick \ldots\ tick);
global macros as \texttt{\$name}.

\begin{lstlisting}
local x = 5
display "x = `x'"             // x = 5

global path "/home/me/data"
use "$path/employment.dta", clear

local vars "gdp pop unemp"
foreach v of local vars {
    summarize `v'
}
\end{lstlisting}

After most estimation commands, results are stored in the \texttt{e()}
namespace:

\begin{lstlisting}
regress y x1 x2
display e(N)         // sample size
display e(r2)        // R-squared
display _b[x1]       // x1's coefficient
display _se[x1]      // x1's standard error
\end{lstlisting}

Summarize stores results in \texttt{r()}:

\begin{lstlisting}
summarize gdp
display r(mean)
display r(sd)
\end{lstlisting}

\subsection{Control flow}

\begin{lstlisting}
* Conditional
if `n' > 100 {
    display "large sample"
}
else {
    display "small sample"
}

* Foreach over a list, varlist, or numlist
foreach v of varlist gdp pop unemp {
    quietly summarize `v'
    display "`v': N = " r(N) ", mean = " r(mean)
}

* Forvalues
forvalues i = 1/5 {
    display "iteration `i'"
}
\end{lstlisting}

\subsection{Capture and quietly}

\texttt{quietly} suppresses output; \texttt{capture} additionally
suppresses errors (storing the return code in \texttt{\_rc}):

\begin{lstlisting}
quietly regress y x       // runs, no output
display _b[x]              // coefficient is still available

capture confirm variable z // doesn't error if z doesn't exist
if _rc != 0 {
    display "z is missing"
}
\end{lstlisting}

\subsection{The \texttt{by:} prefix}

\begin{lstlisting}
sort country year                // required before by:
by country: generate dy = y - y[_n-1]

bysort country (year): generate y_first = y[1]   // sorts first
\end{lstlisting}

\texttt{tea} enforces the sort requirement — using \texttt{by:} on
unsorted data is a hard error, not a silent miscomputation.  Use
\texttt{bysort} when you want it to sort automatically.

\section{Data management}

\subsection{Importing data}

\texttt{tea} reads CSV, TSV, Stata \texttt{.dta}, Excel \texttt{.xlsx},
and OpenDocument \texttt{.ods} files, plus its own native \texttt{.tea}
binary format.

\begin{lstlisting}
import delimited "data.csv", clear      // CSV
import delimited "data.tsv", clear      // TSV (auto-detected)
import excel "data.xlsx", firstrow clear
import excel "data.xlsx", sheet("Sheet2") firstrow clear

use "data.dta", clear                   // Stata format
use "data.tea", clear                   // native tea format
\end{lstlisting}

\texttt{tea} auto-detects the delimiter for \texttt{.csv} and
\texttt{.tsv}; if the file extension is ambiguous, pass
\texttt{delimiter(",")} or \texttt{delimiter(tab)} explicitly.

The \texttt{firstrow} option (for Excel) tells \texttt{tea} to use the
first row as variable names.  Without it, columns are named \texttt{A},
\texttt{B}, etc.

CSV import handles RFC 4180–style quoted fields, including embedded
commas and newlines.

\subsection{Saving data}

\begin{lstlisting}
save "out.dta", replace            // Stata format (default for .dta)
save "out.tea", replace            // native tea format
export delimited "out.csv", replace
export delimited "out.tsv", replace
\end{lstlisting}

\textbf{Format choice.}  Use \texttt{.dta} if the data will be read by
Stata.  Use \texttt{.tea} if it's a temporary file between \texttt{tea}
sessions (faster to read/write, no readstat overhead).  Use \texttt{.csv}
or \texttt{.tsv} if a non-statistical tool will consume the data.

\subsection{Generating and modifying variables}

\begin{lstlisting}
generate gdp_log = log(gdp)
generate growth = (gdp - gdp[_n-1]) / gdp[_n-1]
replace growth = . if year == 1990             // missing for first year

generate str8 region = "WEST"
replace region = "EAST" if country_id == "CHN"
\end{lstlisting}

The \texttt{str\#} prefix declares a string variable (the number is the
maximum width, used by Stata when writing \texttt{.dta} files; in
\texttt{tea} it's cosmetic on read but compressed back on write).

\subsection{\texttt{egen} extended generators}

\texttt{egen} provides aggregate functions that \texttt{generate} can't
do in a single expression:

\begin{lstlisting}
egen mean_gdp     = mean(gdp), by(country)
egen total_pop    = total(pop)
egen rank_gdp     = rank(gdp)
egen group_id     = group(country region)         // assigns 1..K
egen row_total    = rowtotal(x1 x2 x3 x4)
egen has_any      = anyvalue(x), values(1 5 9)
\end{lstlisting}

The full list of functions: \texttt{mean}, \texttt{sum}, \texttt{total},
\texttt{count}, \texttt{min}, \texttt{max}, \texttt{sd}, \texttt{var},
\texttt{median}, \texttt{group}, \texttt{rank}, \texttt{rowtotal},
\texttt{rowmean}, \texttt{rowmin}, \texttt{rowmax}, \texttt{rowmiss},
\texttt{rownonmiss}, \texttt{tag}, \texttt{seq}, \texttt{anyvalue},
\texttt{concat}.

\subsection{Type conversion}

\begin{lstlisting}
* Numeric -> string
tostring year, generate(year_str)

* String -> numeric (errors if non-numeric values present)
destring score, generate(score_n)
destring score, generate(score_n) force      // converts non-numeric to .

* String -> integer codes + value label
encode country_id, generate(cid)             // alphabetical 1..K

* Integer codes -> string (inverse of encode)
decode cid, generate(country_str)

* Recoding numeric values
recode age (0/17 = 1 "minor") (18/64 = 2 "working") (65/max = 3 "senior"), gen(age_cat)
\end{lstlisting}

\subsection{Labels}

\begin{lstlisting}
label data "World Economic Outlook subset"
label variable growth "Real GDP growth (%)"

label define region 1 "North" 2 "South" 3 "East" 4 "West"
label values region region
\end{lstlisting}

\section{Exploring data}

\subsection{\texttt{describe}}

\begin{lstlisting}
describe                          // all variables
describe gdp pop                  // just these two
describe gdp*                     // wildcards work
\end{lstlisting}

Shows variable name, storage type, display format, and variable label.

\subsection{\texttt{list}}

\begin{lstlisting}
list                              // all rows (capped at 200)
list in 1/10                      // first 10 rows
list country year gdp if year == 2020
\end{lstlisting}

Column widths auto-fit to the maximum of header and value widths.

\subsection{\texttt{summarize}}

\begin{lstlisting}
summarize                         // all numeric variables
summarize gdp, detail             // adds percentiles, skewness, kurtosis
summarize gdp if year >= 2000
bysort country: summarize gdp
\end{lstlisting}

After \texttt{summarize}, the following \texttt{r()} macros are set:
\texttt{r(N)}, \texttt{r(mean)}, \texttt{r(sd)}, \texttt{r(min)},
\texttt{r(max)}, \texttt{r(sum)}, \texttt{r(Var)}.  With \texttt{detail},
also \texttt{r(p1)}, \texttt{r(p5)}, \texttt{r(p10)}, \texttt{r(p25)},
\texttt{r(p50)}, \texttt{r(p75)}, \texttt{r(p90)}, \texttt{r(p95)},
\texttt{r(p99)}, \texttt{r(skewness)}, \texttt{r(kurtosis)}.

\subsection{\texttt{tabulate}}

\begin{lstlisting}
tabulate region                                 // one-way frequency table
tabulate region year, row column cell           // two-way + percentages
tabulate region, summarize(gdp) means           // means of gdp by region
\end{lstlisting}

\subsection{\texttt{codebook}}

\begin{lstlisting}
codebook                          // all variables
codebook gdp                      // just one
\end{lstlisting}

Reports type, unique values, missing count, range, and distribution
summary.  More compact than \texttt{summarize, detail} for screening
a new dataset.

\subsection{\texttt{count}}

\begin{lstlisting}
count                             // total observations
count if missing(gdp)
count if gdp > 0 & year >= 2000
\end{lstlisting}

After \texttt{count}, \texttt{r(N)} holds the count.

\section{Reshaping, merging, aggregating}

\subsection{\texttt{sort} and \texttt{gsort}}

\begin{lstlisting}
sort country year                 // ascending on both
gsort -gdp                        // descending on gdp
gsort country -year               // mixed
\end{lstlisting}

\subsection{\texttt{reshape}}

\texttt{reshape} is in-place — to keep the original you must
\texttt{frame copy} first.

\begin{lstlisting}
* Wide to long: each yY variable becomes y at year Y
reshape long y, i(country) j(year)

* Long to wide
reshape wide y, i(country) j(year)

* Multiple stubs
reshape long y z, i(country) j(year)
\end{lstlisting}

\subsection{\texttt{merge}}

\begin{lstlisting}
merge 1:1 country year using "macro.dta"
merge m:1 country     using "country_attrs.dta"
merge 1:m country year using "panel_history.dta"
\end{lstlisting}

After \texttt{merge}, the variable \texttt{\_merge} encodes the match
status: 1 = master only, 2 = using only, 3 = matched.

\texttt{m:m} is deliberately rejected.  If you think you need
\texttt{m:m}, you almost certainly want a different operation
(\texttt{joinby}-style cross product, or an explicit deduplication
before merging).

\subsection{\texttt{collapse}}

\begin{lstlisting}
collapse (mean) gdp pop (sum) revenue (sd) sd_gdp = gdp, by(country)
collapse (median) gdp [aweight=pop], by(region year)
\end{lstlisting}

Available aggregators: \texttt{mean}, \texttt{median}, \texttt{sum},
\texttt{min}, \texttt{max}, \texttt{sd}, \texttt{count}, \texttt{first},
\texttt{last}, \texttt{p25}, \texttt{p50}, \texttt{p75}, \texttt{p10},
\texttt{p90}.

\subsection{Frames}

A frame is a named in-memory dataset.  Multiple frames let you keep
several datasets loaded at once.

\begin{lstlisting}
frame create macro
frame change macro
use "macro.dta", clear

frame change default
* now we're back on the original frame

frame copy default backup    // duplicate default into a new frame
frame drop backup
\end{lstlisting}

\section{Time series and panel data}

\subsection{Declaring panel / time structure}

\begin{lstlisting}
tsset year                        // pure time series
xtset country year                 // panel
xtset country year, yearly         // explicit time unit
\end{lstlisting}

After this, time-series operators are valid:

\begin{center}
\begin{tabular}{ll}
\toprule
Operator & Meaning \\
\midrule
\texttt{L.x} & first lag (one period back) \\
\texttt{L2.x} & second lag \\
\texttt{L(1/3).x} & first through third lags \\
\texttt{F.x} & first lead \\
\texttt{D.x} & first difference \\
\texttt{D2.x} & second difference (Stata's notation) \\
\texttt{S.x} & seasonal difference \\
\bottomrule
\end{tabular}
\end{center}

These can mix with factor variables (with one limitation; see
\S\ref{sec:factor}):

\begin{lstlisting}
regress y L.y x i.region              // OLS with lag and region dummies
xtreg y L(1/3).y x, fe                 // panel FE with three lags
\end{lstlisting}

\texttt{tea}'s time-series operators are \emph{gap-aware}: if a country
has data for 1995 but not 1996, \texttt{L.y} for 1997 returns missing
(not the 1995 value).

\subsection{\texttt{xtdescribe}}

\begin{lstlisting}
xtdescribe
\end{lstlisting}

Reports number of panels, observations per panel (min/avg/max), and
sample pattern.  Useful for checking whether a panel is balanced.

\section{Factor variables}\label{sec:factor}

Stata's factor-variable grammar lets you include categorical regressors
and their interactions inline, without manually generating dummies.
\texttt{tea} supports the same grammar across every estimator
(\texttt{regress}, \texttt{xtreg}, \texttt{logit}, \texttt{probit},
\texttt{poisson}, \texttt{ivregress}).

\subsection{Atoms}

\begin{center}
\begin{tabular}{ll}
\toprule
Token & Meaning \\
\midrule
\texttt{i.var} & dummies for each non-base level; base = smallest level \\
\texttt{ib2.var} & same, with base level explicitly set to 2 \\
\texttt{c.var} & continuous regressor (mostly for use inside \texttt{\#}) \\
\bottomrule
\end{tabular}
\end{center}

\begin{lstlisting}
regress wage exper i.region              // region dummies
regress wage exper ib2.region            // base = region 2
\end{lstlisting}

\texttt{i.var} requires \texttt{var} to be numeric.  For string
identifiers (e.g.\ ISO country codes), use \texttt{encode} first:

\begin{lstlisting}
encode country_id, gen(cid)
regress y x i.cid
\end{lstlisting}

\subsection{Interactions}

\begin{center}
\begin{tabular}{ll}
\toprule
Token & Meaning \\
\midrule
\texttt{A\#B} & cross product: $(K_A - 1) \times (K_B - 1)$ columns \\
\texttt{A\#\#B} & main effects + interaction (equivalent to \texttt{A B A\#B}) \\
\bottomrule
\end{tabular}
\end{center}

\begin{lstlisting}
regress wage i.region#c.exper            // region-specific slopes
regress wage i.region##c.exper           // adds main effects of region and exper
regress wage i.region#i.year             // region-by-year interaction
\end{lstlisting}

The \texttt{c.} prefix tells \texttt{tea} to treat the variable as
continuous in an interaction — without it, a numeric variable would be
expanded as a factor.

\subsection{Coefficient naming}

Factor expansions produce coefficients with structured names that
\texttt{predict}, \texttt{margins}, and \texttt{\_b[]} all understand:

\begin{lstlisting}
regress wage i.region
display _b[2.region]                      // coefficient on region 2 dummy
display _b[3.region#c.exper]              // interaction coefficient
\end{lstlisting}

\subsection{Known limitation: factor × TS-op composition}

Tokens like \texttt{i.country\#c.L.gdp} (factor interaction with a
lagged regressor) are \textbf{not} supported in v\teaversion.

\textbf{Workaround:} pre-compute the lag manually and use the bare
variable inside the interaction:

\begin{lstlisting}
generate L1_gdp = L.gdp
regress y i.country#c.L1_gdp                // works
\end{lstlisting}

\section{Linear regression}

\subsection{Basic syntax}

\begin{lstlisting}
regress y x1 x2 x3
regress y x1 x2, vce(robust)               // HC1 robust SEs
regress y x1 x2, vce(cluster country)      // CR1 clustered SEs
regress y x1 x2 [fweight=n], vce(robust)   // frequency weights
\end{lstlisting}

Weight types: \texttt{fweight} (frequency, integer), \texttt{aweight}
(analytic, rescaled to sum to $N$), \texttt{pweight} (probability,
no rescaling), \texttt{iweight} (importance, like \texttt{aweight} but
no rescaling).

\subsection{Output}

A standard ANOVA table with $F$-statistic and $R^2$, followed by the
coefficient table with $t$-statistics, $p$-values, and 95\% confidence
intervals.  After \texttt{regress}, the following are stored:

\begin{itemize}
\item \texttt{e(N)}, \texttt{e(r2)}, \texttt{e(r2\_a)}, \texttt{e(rmse)},
  \texttt{e(F)}, \texttt{e(p)}, \texttt{e(df\_r)}, \texttt{e(df\_m)}
\item \texttt{\_b[varname]} and \texttt{\_se[varname]} for each
  coefficient, plus \texttt{\_b[\_cons]} and \texttt{\_se[\_cons]}.
\end{itemize}

\subsection{Robust and clustered SEs}

The implementation uses HC1 for robust and CR1 for clustered, matching
Stata's defaults.  Cluster variables may be string (panel IDs) or
numeric.

\subsection{Constraints and hypothesis tests}

After \texttt{regress}, use \texttt{test} for joint hypotheses and
\texttt{lincom} for linear combinations:

\begin{lstlisting}
regress y x1 x2 x3 x4

test x1 = x2                                  // single restriction
test x1 = 0, x2 = 0                           // joint (F-test)
test x1 x2 x3                                 // all = 0 jointly

lincom x1 + x2                                 // linear combination + CI
lincom 0.5*x1 + 0.5*x2 - x3
\end{lstlisting}

\section{Panel regression}

\subsection{Fixed effects}

\begin{lstlisting}
xtset country year
xtreg y x1 x2, fe                              // within (default for xtreg)
xtreg y x1 x2, fe vce(cluster country)
\end{lstlisting}

Output includes:
\begin{itemize}
\item Within / between / overall $R^2$
\item $\sigma_u$, $\sigma_e$, $\rho$ (variance components)
\item $F$-test of $u_i = 0$
\item \texttt{corr(u\_i, Xb)} — informally measures the FE--RE departure
\end{itemize}

Identifies as \texttt{e(cmd) = "xtreg"} with \texttt{e(model) = "fe"}.

\subsection{Random effects}

\begin{lstlisting}
xtreg y x1 x2, re                              // Swamy-Arora FGLS
\end{lstlisting}

Implementation: Swamy-Arora variant (the Stata default).  Two-step
FGLS — first estimate $\sigma_u^2$, $\sigma_e^2$ from the within and
between residuals; then quasi-demean using
$\theta = 1 - \sigma_e / \sqrt{T \sigma_u^2 + \sigma_e^2}$.

\subsection{Between effects}

\begin{lstlisting}
xtreg y x1 x2, be                              // OLS on panel means
\end{lstlisting}

Collapses the data to one observation per panel (the panel means of
$y$ and each $x$), then runs OLS.  This is the BE estimator that
underpins one of the components of the RE-FE decomposition.

\subsection{Hausman test}

\begin{lstlisting}
quietly xtreg y x1 x2, fe                      // populates fe_est
quietly xtreg y x1 x2, re                      // populates re_est
hausman                                         // tests FE vs RE
\end{lstlisting}

Order of \texttt{xtreg fe} and \texttt{xtreg re} doesn't matter;
\texttt{tea} keeps both in dedicated workspace slots and the
\texttt{hausman} command (no arguments needed) tests their
coefficient difference.

\section{Maximum likelihood estimators}

\texttt{tea} implements logit, probit, and Poisson regression via a
shared Newton-Raphson driver.  All three support \texttt{if}, \texttt{in},
weights, and \texttt{vce(robust|cluster)}.

\subsection{\texttt{logit}}

\begin{lstlisting}
logit voted income education
logit y x1 x2 i.region, vce(cluster country)
\end{lstlisting}

Reports the maximum likelihood iteration trace, LR $\chi^2$ test
against the constant-only model, pseudo $R^2$, and the coefficient
table with $z$-statistics.

\subsection{\texttt{probit}}

Same syntax, normal CDF instead of logistic CDF:

\begin{lstlisting}
probit y x1 x2 i.region
\end{lstlisting}

\subsection{\texttt{poisson}}

For count outcomes:

\begin{lstlisting}
poisson accidents speed_limit population, vce(cluster state)
\end{lstlisting}

The link is $\log E[y \mid x] = x'\beta$; coefficients are
log-rate-ratios.  $y < 0$ is rejected.  Non-integer $y$ is permitted
(treated as the GLM extension).

\textbf{Important:} \texttt{tea}'s Poisson drops the $\log(y!)$ constant
in the log-likelihood, so the absolute \texttt{e(ll)} value differs from
Stata's, but the coefficient estimates, SEs, and likelihood-ratio
tests are identical.

\subsection{Marginal effects via \texttt{margins}}

After any of the above, \texttt{margins} produces average or
at-the-mean marginal effects:

\begin{lstlisting}
logit y x1 x2 i.region
margins, dydx(*)                              // average marginal effects
margins, dydx(x1)                             // just x1
margins, dydx(*) atmeans                      // at-the-means MEs
\end{lstlisting}

Standard errors use the delta method.

\section{Instrumental variables}

\texttt{ivregress 2sls} implements two-stage least squares with a
first-stage $F$ diagnostic for weak instruments.

\subsection{Syntax}

\begin{lstlisting}
ivregress 2sls y [exog_vars] (endog_vars = instruments) [if] [in] [weight] [, options]
\end{lstlisting}

The parenthesized group is the key piece: variables to the left of
the \texttt{=} are endogenous; variables to the right are excluded
instruments.

\subsection{Example}

\begin{lstlisting}
* Wage equation: education is endogenous, parents' education is the IV
ivregress 2sls wage exper exper2 (educ = momeduc dadeduc), vce(robust)
\end{lstlisting}

Output includes the coefficient table (with $z$-statistics from
asymptotic normality), the list of instrumented variables, the list
of instruments (included exogenous + excluded), and the first-stage
$F$-statistic for the weakest endogenous regressor.

\subsection{Interpreting the first-stage $F$}

A common rule of thumb (Stock-Yogo): if the first-stage $F$ is below
10, the instrument is weak and the IV estimates may have substantial
finite-sample bias.  \texttt{tea} flags this in the output with a
caret.

\subsection{Variance estimators}

\begin{lstlisting}
ivregress 2sls y (x = z)                              // classical
ivregress 2sls y (x = z), vce(robust)                 // HC1
ivregress 2sls y (x = z), vce(cluster country)        // CR1
\end{lstlisting}

The sandwich form is $V = A \cdot B \cdot A$ where $A = (\hat X'\hat X)^{-1}$
(using fitted endogenous columns) and $B$ uses residuals computed from
the \emph{original} $X$ (this is the correct 2SLS variance — not the
naive OLS-on-fitted variance).

\section{Time series models}\label{sec:tsmodels}

\subsection{ARIMA}

\begin{lstlisting}
tsset year
arima y, arima(p d q) [noconstant]
arima y x1 x2, arima(1 0 1)                   // ARMAX with two exog
\end{lstlisting}

Specifies an ARIMA$(p, d, q)$ model: AR order $p$, difference order $d$,
MA order $q$.  Exogenous regressors can be included (then the model
is ARMAX).

\textbf{Important.}  v\teaversion's \texttt{arima} uses \emph{conditional
likelihood}, not the Kalman-filter exact ML that Stata uses by default.
The differences:

\begin{itemize}
\item Coefficient estimates are consistent and converge to the same
limit, but finite-sample values can differ by a few percent.
\item The absolute log-likelihood differs (we drop the initial-state
contribution).
\item Standard errors come from a numerical-difference Hessian and
can be off by 1--5\% on the MA components specifically.  Point
estimates are unaffected.
\end{itemize}

If you need exact ML, escape to R's \texttt{arima()} or Python's
\texttt{statsmodels.tsa.arima}.

\subsection{Example}

\begin{lstlisting}
* US growth ARIMA(1,0,0)
tsset year
arima growth, arima(1 0 0)
\end{lstlisting}

The output groups coefficients into ``ARMA model'' (constant + exog),
``AR'' ($\phi_i$), ``MA'' ($\theta_j$), and \texttt{/sigma} (the
innovation standard deviation).

\section{Post-estimation}

\subsection{\texttt{predict}}

After any estimator, \texttt{predict} generates a new variable with
the model prediction or a residual:

\begin{lstlisting}
* After regress
regress y x1 x2
predict yhat                          // linear prediction (default: xb)
predict resid, residuals              // residuals
predict resid, r                       // same, shorter

* After logit/probit
logit voted income
predict p                              // Pr(y=1)
predict xb, xb                         // linear index

* After xtreg fe
xtreg y x1, fe
predict u, u                           // estimated u_i
predict e, e                           // idiosyncratic residual
predict ue, ue                         // u_i + e_it
\end{lstlisting}

\subsection{Hypothesis tests}

\texttt{test} performs Wald tests on linear restrictions:

\begin{lstlisting}
regress y x1 x2 x3
test x1 = 0                                    // single restriction
test x1 = 1, x2 = 0                            // joint restriction
test x1 x2 x3                                  // x1 = x2 = x3 = 0
\end{lstlisting}

\texttt{lincom} computes linear combinations with their SEs and 95\% CIs:

\begin{lstlisting}
lincom x1 + x2 - x3
lincom 0.5*x1 + 0.5*x2
\end{lstlisting}

\section{Output and reproducibility}

\subsection{Saving estimates}

The \texttt{estimates} (alias \texttt{est}) command suite saves and
manages named estimation results:

\begin{lstlisting}
regress y x1
estimates store m1

regress y x1 x2
estimates store m2

regress y x1 x2 x3
estimates store m3

estimates dir                          // list stored estimates
estimates table m1 m2 m3, stats(N r2 rmse) star
estimates restore m1                   // load m1 back to last_est
estimates drop m2                      // remove m2
estimates drop _all                    // remove all
\end{lstlisting}

\subsection{\texttt{estout}: publication tables}

For papers, use \texttt{estout} to produce LaTeX, markdown, or plain
side-by-side tables:

\begin{lstlisting}
estout m1 m2 m3                                         // default: LaTeX
estout m1 m2 m3, format(latex) stats(N r2 rmse)
estout m1 m2 m3, format(markdown) se star
estout m1 m2 m3, format(plain) t

estout m1 m2 m3 using "table.tex", stats(N r2)         // write to file
estout m1 m2 m3, title("Regression results") label(tab:main)

* Subset of coefficients
estout m1 m2, keep(x1 x2 _cons) stats(N r2)
estout m1 m2, drop(_cons)                              // drop constant row
\end{lstlisting}

Options:

\begin{center}
\begin{tabular}{ll}
\toprule
Option & Effect \\
\midrule
\texttt{format(latex|markdown|plain)} & output format (default: \texttt{latex}) \\
\texttt{se}, \texttt{t}, \texttt{p} & show SEs / $t$-stats / $p$-values below coefs \\
\texttt{star} & add significance stars (10\%, 5\%, 1\%) \\
\texttt{stats(N r2 r2\_a rmse F \ldots)} & footer rows \\
\texttt{title(...)}, \texttt{label(...)} & LaTeX caption + reference label \\
\texttt{keep(...)}, \texttt{drop(...)} & restrict coefficient rows \\
\texttt{nomtitles}, \texttt{mtitles(...)} & override per-model column labels \\
\texttt{using FILE} & write to file instead of stdout \\
\bottomrule
\end{tabular}
\end{center}

LaTeX output produces a stand-alone \texttt{tabular} block you can
\verb|\input| in your paper.

\subsection{\texttt{log}}

\begin{lstlisting}
log using "session.log", replace
* commands and their output are captured...
log close
\end{lstlisting}

Captures both the commands you ran (echoed with a \texttt{.~} prefix)
and the output they produced.

\section{Random numbers and Monte Carlo}\label{sec:mc}

\subsection{Reproducibility}

\begin{lstlisting}
set seed 12345                                  // fix the PRNG state
\end{lstlisting}

\texttt{tea} uses a 64-bit PCG generator.  \texttt{set seed N} is
deterministic — the same seed always produces the same sequence,
across runs, machines, and \texttt{tea} versions.

\subsection{Functions}

\begin{lstlisting}
generate u = runiform()                         // Uniform[0, 1)
generate z = rnormal()                          // Standard normal
generate w = rnormal(5, 2)                      // Normal(mean=5, sd=2)
\end{lstlisting}

Normal draws use Box-Muller with caching.  All three functions are
available in any expression context (\texttt{generate}, \texttt{replace},
\texttt{egen rowfirst/rowlast}, etc.).

\subsection{A Monte Carlo example}

\begin{lstlisting}
* OLS slope sampling distribution under known DGP
set seed 42
clear
set obs 1000

* True model: y = 2*x + eps,  eps ~ N(0, 1)
generate x = rnormal()
generate eps = rnormal()
generate y = 2*x + eps

regress y x
display "Estimated beta = " _b[x] " (true is 2.0)"
\end{lstlisting}

\section{Escape hatches: when to leave \texttt{tea}}\label{sec:escape}

\texttt{tea} is excellent at the data-prep-to-quick-estimate loop, but
many things are out of scope.  The escape pattern is:

\begin{enumerate}
\item Do data prep in \texttt{tea}.
\item \texttt{export delimited} to CSV.
\item \texttt{shell python myscript.py} (or R, or anything).
\item \texttt{import delimited} the results back.
\end{enumerate}

\subsection{When to escape}

\begin{center}
\begin{tabular}{ll}
\toprule
What you need & Where to go \\
\midrule
Plotting & \texttt{gnuplot}, Python's \texttt{matplotlib}, R's \texttt{ggplot2} \\
GARCH / EGARCH / TGARCH & Python \texttt{arch}, R \texttt{rugarch} \\
Exact-ML ARIMA (Kalman) & R's \texttt{arima()}, \texttt{forecast::Arima} \\
Seasonal ARIMA & R's \texttt{arima()} with \texttt{seasonal} arg \\
Bayesian inference & PyMC, Stan (\texttt{rstan}, \texttt{cmdstanpy}) \\
Mixed-effects models & R's \texttt{lme4}, Python \texttt{statsmodels.MixedLM} \\
Survival analysis & R's \texttt{survival}, Python \texttt{lifelines} \\
Cointegration / VAR / VECM & R's \texttt{urca}, \texttt{vars} \\
Machine learning & \texttt{scikit-learn}, \texttt{xgboost}, \texttt{lightgbm} \\
Network / graph models & \texttt{networkx}, R \texttt{igraph} \\
Spatial econometrics & R's \texttt{spdep}, Python's \texttt{pysal} \\
\bottomrule
\end{tabular}

\section{Known v\teaversion\ limitations}

These are not bugs in the strict sense — they're scope decisions for
v\teaversion\ that we may revisit based on testing.

\subsection{Data and metadata}

\begin{itemize}[leftmargin=*]
\item \textbf{\texttt{.dta} round-trip does not preserve \texttt{xtset}
state.}  After \texttt{save mydata.dta} then \texttt{use mydata.dta},
the panel/time variable assignment is lost.  Workaround: re-run
\texttt{xtset country year} after \texttt{use}.  This is the standard
Stata-do-file convention anyway.
\item \textbf{Variable name length cap of 32 characters.}  Factor-variable
interaction names like \texttt{2010.year\#1985.country\#c.gdp} (28
chars) fit; longer 3-way interactions truncate.
\end{itemize}

\subsection{Econometric estimators}

\begin{itemize}[leftmargin=*]
\item \textbf{ARIMA uses conditional likelihood, not Kalman-filter exact
ML.}  See \S\ref{sec:tsmodels} for details.  For serious work, use R's
\texttt{arima()}.
\item \textbf{No seasonal ARIMA} (\texttt{sar}, \texttt{sma} terms).
\item \textbf{ARIMA MA standard errors} have a 1--5\% precision loss
from the numerical-Hessian computation.  Point estimates are unaffected.
\item \textbf{\texttt{xtreg, be} uses straight OLS} on panel means.
Stata's default applies a slight variance correction for unbalanced
panels (down-weighting groups with few observations).  Differences
appear only on unbalanced panels and only in the SEs (point estimates
are identical).
\item \textbf{No \texttt{xtabond} / system GMM} or other GMM estimators.
\item \textbf{No VAR / VECM.}  Escape to R.
\end{itemize}

\subsection{Factor variables}

\begin{itemize}[leftmargin=*]
\item \textbf{No composition with TS operators.}  Tokens like
\texttt{i.country\#c.L.gdp} are not parsed.  Workaround: pre-compute
the lag with \texttt{gen L1\_gdp = L.gdp} and use \texttt{c.L1\_gdp}.
\item \textbf{\texttt{i.var} requires numeric var.}  Use \texttt{encode}
on string identifiers.
\item \textbf{Cell-count cap of 10\,000.}  Larger factor expansions
are blocked.
\end{itemize}

\subsection{Programming}

\begin{itemize}[leftmargin=*]
\item \textbf{No \texttt{program define}, no \texttt{syntax} parser,
no Mata.}  You can write loops and use macros, but you cannot define
new commands.
\end{itemize}

\subsection{Graphics}

\textbf{No graphics commands.}  Use \texttt{export delimited} and pipe
to \texttt{gnuplot} or to Python / R.

\section{Reporting bugs}

If you find a bug:

\begin{enumerate}
\item Distill it into a small reproducer — a do-file plus a small input
file is ideal.  If the input data is sensitive, replace it with
\texttt{set seed N; gen x = rnormal()} or similar synthetic.
\item Note your \verb|tea --version|, OS, and compiler version.
\item Send to the author at the email associated with your distribution
of \texttt{tea}.  If you don't have one, ask the person who gave you the
software.
\end{enumerate}

What counts as a bug:

\begin{itemize}
\item Wrong numerical answer (with hand-calculated expected value).
\item Crash, segfault, or hang on input that should work.
\item Different behavior from documented Stata semantics (when not in
the ``known limitations'' list above).
\end{itemize}

What does not count as a bug:

\begin{itemize}
\item Missing feature (those go on the v1.1 feature-request list).
\item Disagreement with Stata's exact numerical output to many decimal
places (small differences arise from BLAS vendor variation and are not
correctness issues).
\item Things listed in \S20 above as known limitations.
\end{itemize}

\section{Stata cheat sheet}

A quick reference for Stata users adapting to \texttt{tea}.

\subsection*{Same as Stata}

\begin{itemize}[leftmargin=*]
\item Syntax: \texttt{[prefix:] command [varlist] [if] [in] [weight] [, options]}
\item Missing-value algebra, including \texttt{if x > 5} including missing
\item Macros: \verb|`local'| and \texttt{\$global}, \texttt{r()} and
\texttt{e()} returns
\item \texttt{by:}, \texttt{bysort:}, the \texttt{[\_n-1]} subscript,
\texttt{\_N}
\item Value labels: \texttt{label define / values / list}
\item Factor variables (\texttt{i.}, \texttt{c.}, \texttt{ib<n>.},
\texttt{\#}, \texttt{\#\#})
\item Time-series operators (\texttt{L. F. D. S.})
\end{itemize}

\subsection*{Tea-specific differences}

\begin{itemize}[leftmargin=*]
\item \texttt{m:m} merge is rejected with an explanatory message.
\item \texttt{xtreg, be} uses simple OLS on panel means.
\item \texttt{arima} uses conditional likelihood.
\item \texttt{i.country\#c.L.gdp} doesn't parse; pre-compute the lag.
\item No \texttt{.gph} graphics — use \texttt{export} and external tools.
\item \texttt{program define}, \texttt{syntax}, and Mata are absent.
\end{itemize}

\subsection*{Stata commands that don't exist in tea}

\begin{itemize}[leftmargin=*]
\item Graphics: \texttt{graph}, \texttt{twoway}, \texttt{histogram}, etc.
\item GMM: \texttt{gmm}, \texttt{xtabond}, \texttt{xtdpd}.
\item Time series: \texttt{var}, \texttt{vec}, \texttt{dfgls},
\texttt{dfuller}, \texttt{xcorr}, \texttt{wntestq}.
\item Survival: \texttt{stcox}, \texttt{streg}, \texttt{stset}.
\item Mixed: \texttt{mixed}, \texttt{xtmixed}, \texttt{meqrlogit}.
\item Bayesian: \texttt{bayes:}, \texttt{bayesmh}.
\item Multivariate: \texttt{mvreg}, \texttt{factor}, \texttt{pca}.
\item Multiple imputation: \texttt{mi}.
\end{itemize}

For each of these, see \S\ref{sec:escape} for the recommended external
tool.
